Hexo使用方法

Hexo的详细使用文档详见这里。

Hexo是一个博客框架，可以将本地的.md文档渲染成静态网页并布署到github。

Hexo的安装

  • Node.js环境 (因为Hexo使用Node.js编写，所以必须安装这个环境)

  • Git环境 (因为需要与Github互相沟通，所以必须安装)

    具体见此处说明。

  • Hexo本身的安装

    $ npm install -g hexo-cli

    npm是Node package manager, -g是全局安装

    此句的意思是使用npm全局安装hexo-cli这个软件。

Hexo建站

$ hexo init <folder>	#在要建博客的文件夹用hexo初始化，本质是从某个github仓库里拉取文件
$ cd <folder>			#进入该文件夹
$ npm install			#"npm install"是Node Package Manager(npm)的一个命令，用于在项目中安装所需的依赖包。它会读取项目目录中的"package.json"文件，并从npm registry下载定义的依赖包。在项目中使用依赖包的代码之前，通常需要先运行"npm instal"命令。

hexo init完成后，指定文件夹的目录如下：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

package.json的内容为

{
"name": "hexo-site",
  "version": "0.0.0",
  "private": true,
  "scripts": {								
    # 定义调用函数的名称，也即只有运行了npm install之后，你再键入 hexo g才会运行相应函数
    "build": "hexo generate",
    "clean": "hexo clean",
    "deploy": "hexo deploy",
    "server": "hexo server"
  },
  "hexo": {
    "version": ""
  },
  "dependencies": {							
    # 定义依赖关系
    "hexo": "^6.3.0",
    "hexo-generator-archive": "^2.0.0",
    "hexo-generator-category": "^2.0.0",
    "hexo-generator-index": "^3.0.0",
    "hexo-generator-tag": "^2.0.0",
    "hexo-renderer-ejs": "^2.0.0",
    "hexo-renderer-marked": "^6.0.0",
    "hexo-renderer-stylus": "^3.0.0",
    "hexo-server": "^3.0.0",
    "hexo-theme-landscape": "^1.0.0"
  }
}

其中的scaffolds脚手架目录是模板目录，以后新建的文章

scaffolds
├── draft.md		#当运行$hexo new draft xxx时使用此模板
├── page.md		#当运行$hexo new Page xxx时使用此模板
└── post.md		#当运行$hexo new post xxx时使用此模板

其中的source文件夹存放你建好的页面，后面当运行

$ hexo g时会把此文件夹中的页面渲染到.\public文件夹

source
├── _posts		#存放文章
|   ├── 2023-05-19-MarkdownLearning.md

├── img			#存放网页中用到的图片
├── list		#博客的列表
└── support		#博客的页尾

Hexo的配置

对Hexo的配置主要是对_config.yml文件的配置。

Hexo默认不使用主题，当使用主题时需要做以下修改：

# Extensions
theme: A4	#默认是false		
# 本博客采用的主题名为A4，当安装主题后，网站根目录下的_config.a4.yml里的配置文件会代替或补充_config.yml文件中的相应参数

可以说配置是使用Hexo建立网站的核心，你必须理解里面的各个参数才可能做到对网站自由的修改，而要实现这个目的，需要花一定时间仔细阅读说明文档，如果应用了主题，还要阅读主题的说明文档，麻烦的是，很多时候读了也未必能明确某个参数是什么意思，这时需要补充做一些测试才行。

Hexo的主要指令

_config.yml文件中有以下参数：

# Writing
default_layout: post

新建页面

# 新建一篇文章。如果没有设置 [layout] 的话，默认使用 _config.yml中的default_layout参数代替。
$ hexo new [layout] <title>

# 所以如果写成这样,命令的含义是一样的
$ hexo new <title>
# 如果标题包含空格的话，请使用引号括起来，像这样
$ hexo new "this is a post"

# 新建一篇post文章,程序会到scaffolds文件夹下寻找模板post.md,建立title.md并保存到source文件夹中
$ hexo new post <titile>

# 新建一篇page文章,程序会到scaffolds文件夹下寻找模板page.md,建立title.md并保存到source文件夹中
$ hexo new page <titile>

# 新建一篇draft文章,程序会到scaffolds文件夹下寻找模板page.md,建立title.md并保存到source文件夹中
$ hexo new draft <titile>

# 新建一篇photo文章,程序会到scaffolds文件夹下寻找模板photo.md,建立title.md并保存到source文件夹中
$ hexo new photo <titile>

_config.yml文件中有以下参数控制生成的网页名：

# Writing
#默认生成"title.md"这样的文件名
new_post_name: :title.md
#若想生成"2023-05-19-title.md"这样的文件名可以修改参数为
new_post_name: :year-:month-:day-:title.md

clean

$ hexo clean

清除缓存文件 (db.json) 和已生成的静态文件目录 (\public)。

generate

$ hexo generate
# generate可以简化成g
$ hexo g

生成静态文件，并保存到public文件夹中。

deploy

$ hexo deploy
# deploy可以简化成d
$ hexo d

将public文件夹下的静态页面部署网站到github。

上述命令可以也合并执行：

$ hexo clean && hexo g && hexo d

server

$ hexo server
# sever可以简化成s
$ hexo s

本地启动服务器。默认情况下，访问网址为： http://localhost:4000/。

主要的指令就是这几条，更完整的指令详见说明文档。
