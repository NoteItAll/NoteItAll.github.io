Markdown标记语言

Markdown标记语言使人们可以摆脱杂乱的html文本，对于简单的个人博客类静态网页来说非常适用，相应的编辑软件Typora更是把这种极简风格发挥得淋漓尽至，在书写过程中甚至不太能感到自己在编辑网页。

Markdown的基本语法也是很简单易学的。这里是Markdown的官方教程文档。

标题

1-6级标题分别为在文字前面添加1到6个# 号，文字前要留一个英文空格。

# 1级标题
## 2级标题
### 3级标题
#### 4级标题
##### 5级标题
###### 6级标题

粗体（Bold）

在要加粗的字两侧加两个星号 ** 或两个下划线 __

这是**重要**的内容
这是 __重要__ 的内容 #因为每个平台不同可能需要在前后加一个空格，但这样渲染之后会有空格，还是建议用**保险

这是 重要 的内容

文字前加 > ,可以加很多级。

> 被引用的内容
>> 二级引用的内容
>>> 三级引用的内容

有序列表

直接以1. 开始一句话，用制表符tab可以开启次一级列表。

1. 我先做饭
2. 我再吃饭
    1. 我先吃肉
    2. 我不吃菜
3. 我后洗碗

  1. 我先做饭

  2. 我再吃饭

    1. 我先吃肉
    2. 我不吃菜

  3. 我后洗碗

无序列表

以 *或 +或 - 引导文本，这些符号可以混用。

* 我先做饭
+ 我再吃饭
    - 我先吃肉
    + 我不吃菜
+ 我后洗碗
  • 我先做饭
  • 我再吃饭
    • 我先吃肉
    • 我不吃菜
  • 我后洗碗

简单局部代码

把代码以单上标括起来

I need to `print` this page.

I need to print this page.

把代码以双上标括起来，注意不是双引号，这样内部也出现单上标符号。

I need to ``print `this` page``.

围栏式代码块

连续三个单上标```或三个波浪号 ~~~，指定所用的语言后可以支持语法高亮。

如果使用了hexo或应用了某种主题则需要在 _config.yml中打开语法高亮的设置。

highlight: 
    enable: false #要高亮的话要改成 true

对应的HTML代码：<a href="超链接地址" title="超链接title">超链接显示名</a>

这是一个[超链接](https://www.google.com)。

<fake@example.com>

不同的 Markdown 应用程序处理URL中间的空格方式不一样。为了兼容性，请尽量使用%20代替空格。

[https://www.example.com/my great page](https://www.example.com/my%20great%20page)

图片

插入图片Markdown语法代码：![图片alt](图片链接 "图片title")。

对应的HTML代码：<img src="图片链接" alt="图片alt" title="图片title">

在hexo中图片只能放在网站所在目录\source\img，不可以放在网站所在目录\source\_posts\img，否则会找不到图片。

图片较少时放在img文件夹即可，如果图较多，建义放在图床内，推荐使用picGo。

自动网址链接

许多Markdown处理器会自动将URL转换为链接。这意味着如果您输入http://www.example.com ，即使您未使用方括号，您的Markdown处理器也会自动将其转换为链接。如果您不希望自动链接URL，则可以通过将URL表示为带反引号的代码来删除该链接。

`http://www.example.com`

呈现的输出如下所示：

http://www.example.com
